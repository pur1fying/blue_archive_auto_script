이벤트 스캔 작성 안내 (남은 횟수 예산 모델):

「이벤트 스테이지 스캔 스테이지」와 「이벤트 스테이지 스캔 남은 횟수」는 모두 쉼표로 구분하며 순서대로 짝지어 입력합니다.

남은 횟수는 각 스테이지의 남은 스캔 횟수를 기록하며 실행마다 누적 차감됩니다: 스캔 성공 시 실제 스캔한 횟수만큼 차감하여 저장하며, 0이 되면 해당 스테이지를 건너뜁니다.

-1은 해당 스테이지를 마지막에 스캔하며 횟수 무제한(현재 체력이 허용하는 만큼)을 의미합니다. -1은 차감되지 않습니다.

이번 실행의 실제 스캔 횟수 = min(남은 횟수, 현재 체력 / 1회당 체력 소모); -1 스테이지 = 현재 체력 / 1회당 소모.

여러 스테이지 스캔:

스테이지와 남은 횟수를 쉼표로 구분해 짝지으며, 유한 횟수 스테이지를 먼저(작성 순서대로), -1 스테이지를 그 다음에 스캔합니다.

예:

스캔 스테이지: 9, 10, 11

남은 횟수: 25, 3, -1

의미: 9번 스테이지(최대 25회, 차감 후 저장), 10번 스테이지(최대 3회)를 차례로 스캔한 뒤, 11번 스테이지를 마지막에 무제한으로 스캔합니다.

다음 실행 시 9번·10번 스테이지는 남은 횟수에서 이어 차감되며, 11번 스테이지는 계속 -1입니다.
